Algoritmo do Servidor Central

Este é o mais simples dos algoritmos para obter exclusão mútua em um sistema distríbuido, pois ele utiliza um para gerenciar os acessos na seção critica. Seja P, um conjunto de processos Pi, com 0 ≤ i < n, onde n é o número de processos participantes da computação, cada processo Pi que deseja entrar na seção critica deve possuir de acesso. O token pode ser obtido através do envio de uma solicitação para o servidor central. O servidor por sua vez que as respostas do servidor representam o envio de uma ficha (token), que neste modelo significa permissão para entrar na seção crítica.

O gerenciamento de acesso da sessão crítica funciona da seguinte forma:

  • Se o servidor estiver com o token, ele responde a primeira solicitação que recebeu, enviando o token
  • Se o servidor não estiver com o token ele não responderá e irá enfileirar as solicitações, em uma fila local. Pois algum processo Pi está com o token.
